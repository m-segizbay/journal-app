## Описание работы:

Практическая работа состоит из **двух этапов**:

### **ЭТАП 1 (Неделя 1): Управление студентами (CRUD операции)**
**Файл:** `journal_part1.py`

На первом этапе студенты создают приложение для управления списком студентов с возможностью:
- Добавления новых студентов
- Удаления студентов из системы
- Редактирования информации о студентах
- Просмотра полного списка студентов в таблице

**Структура базы данных:**

```
Таблица: students
┌─────┬─────────────────┬─────────────────────┐
│ id  │ fio             │ created_at          │
├─────┼─────────────────┼─────────────────────┤
│ 1   │ Иван Сидоров    │ 2026-03-10 10:30:00 │
│ 2   │ Мария Петрова   │ 2026-03-10 10:35:00 │
│ 3   │ Петр Иванов     │ 2026-03-10 10:40:00 │
└─────┴─────────────────┴─────────────────────┘
```

**Описание полей таблицы:**

| Поле | Тип данных | Описание |
|------|-----------|---------|
| id | INTEGER PRIMARY KEY AUTOINCREMENT | Уникальный идентификатор студента |
| fio | TEXT (UNIQUE) | Фамилия и имя студента (не может быть дублей) |
| created_at | TIMESTAMP DEFAULT CURRENT_TIMESTAMP | Дата и время добавления студента |

**Функциональность:**
- ✅ Добавление новых студентов
- ✅ Удаление студентов из базы
- ✅ Редактирование данных студентов
- ✅ Просмотр списка в виде таблицы
- ✅ Проверка корректности ввода (ФИО не может быть дублем)

### **ЭТАП 2 (Неделя 2): Полная система управления оценками**
**Файл:** `journal_complete.py`

На втором этапе добавляется функциональность для работы с оценками:

**Структура базы данных:**

```
Таблица 1: students (как в этапе 1)

Таблица 2: grades
┌─────┬──────────────┬──────┬────────┐
│ id  │ student_id   │ week │ grade  │
├─────┼──────────────┼──────┼────────┤
│ 1   │ 1            │ 1    │ 85     │
│ 2   │ 1            │ 2    │ 90     │
│ 3   │ 1            │ 3    │ н.я.   │
│ 4   │ 2            │ 1    │ 75     │
└─────┴──────────────┴──────┴────────┘
```

**Описание полей таблицы grades:**

| Поле | Тип данных | Описание |
|------|-----------|---------|
| id | INTEGER PRIMARY KEY | Уникальный идентификатор записи оценки |
| student_id | INTEGER FOREIGN KEY | Связь с таблицей students |
| week | INTEGER | Номер недели (1-15) |
| grade | TEXT | Оценка (0-100, "н.я.", "н.п.") |

**Дополнительная функциональность:**
- ✅ Добавление оценок студентам
- ✅ Редактирование оценок (двойной клик на ячейку)
- ✅ Автоматический расчет РК1 (средняя за недели 1-7)
- ✅ Автоматический расчет РК2 (средняя за недели 8-14)
- ✅ Автоматический расчет итоговой оценки: (РК1 + РК2) / 2
- ✅ Поддержка специальных значений:
  - **н.я.** (не явился) — считается как 0
  - **н.п.** (не предусмотрено) — не учитывается в расчетах

---

## Варианты оценок и их обработка:

| Значение | Описание | Обработка |
|----------|---------|-----------|
| 0-100 | Числовая оценка | Учитывается как есть |
| н.я. | Не явился | Считается как 0 в расчетах |
| н.п. | Не предусмотрено | Исключается из расчета |
| (пусто) | Оценка не выставлена | Исключается из расчета |

**Пример расчета РК1 для студента:**
```
Недели 1-7: 85, 90, н.я., 75, 80, н.п., 88
Учитываемые: 85, 90, 0 (н.я.), 75, 80, 88
РК1 = (85 + 90 + 0 + 75 + 80 + 88) / 6 = 83.17 ≈ 83
```

## Ключевые понятия:

### **SQLite и Python**
- `sqlite3.connect()` — подключение к БД
- `cursor.execute()` — выполнение SQL запросов
- `conn.commit()` — сохранение изменений
- `FOREIGN KEY` — связь между таблицами

### **Tkinter компоненты**
- `ttk.Treeview` — таблица для отображения данных
- `tk.Toplevel` — диалоговые окна
- `messagebox` — окна сообщений об ошибках и успехе

### **SQL операции**
```sql
-- CREATE (добавление)
INSERT INTO students (fio) VALUES (?)

-- READ (чтение)
SELECT * FROM students WHERE id = ?

-- UPDATE (обновление)
UPDATE students SET fio = ? WHERE id = ?

-- DELETE (удаление)
DELETE FROM students WHERE id = ?
```
